# Multi-Level Bitmap + Incremental Counter

## 三层位域

```
GB区: 1 bit per 1GB  → 1GB 文件 = 1 byte
MB区: 1 bit per 1MB  → 1GB 文件 = 128 bytes  
4K区: 1 bit per 4KB  → 1GB 文件 = 32 KB
```

## 核心字段

```rust
struct Search {
    has_match_4k: Vec<u8>,   // 4KB 页位域
    has_match_mb: Vec<u8>,   // 1MB 块位域
    has_match_gb: Vec<u8>,   // 1GB 块位域
    match_count: usize,      // 累计匹配数（增量）
    scanned: usize,          // 已扫描到的偏移
}
```

## 增量扫描逻辑

```
extend(mmap, min_off):
  chunk = 剩余内存可承受的 MB 数（如 64MB）
  while scanned < file_size && scanned < min_off + chunk:
    if gb位域=0 && 已扫描过 → 跳过整个GB
    if mb位域=0 && 已扫描过 → 跳过整个MB
    扫描当前1MB块 → 找到匹配 → match_count++
    设置 4k/mb/gb 位域
    scanned += 1MB
```

## 内存估算（1GB 文件）

| 组件 | 大小 |
|------|------|
| 4K位域 | 32 KB |
| MB位域 | 128 B |
| GB位域 | 1 B |
| ranges (10万匹配) | ~400 KB |
| **总计** | **~432 KB** |
