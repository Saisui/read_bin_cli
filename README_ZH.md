# read_bin

[English](README.md) | **中文**

终端 TUI 十六进制查看/编辑器，Rust 实现（v0.1.11）。

**依赖**：ratatui（TUI 框架）、crossterm（终端控制）、memmap2（内存映射文件）、arboard（剪贴板）

### 架构亮点

- **mmap + overlay**：文件内存映射（零拷贝，不 `to_vec`）。编辑存入内存 `HashMap<usize, u8>` overlay — 仅修改的字节被复制，内存与编辑数成正比而非文件大小
- **四级位图搜索**（Saisui 发明，BitSearch）：804 字节固定内存，按需扫描
- **稀疏层级位图**（Saisui 发明）：4K→1MB→1GB→1TB 层级追踪编辑字节，O(1) 查询
- 编辑字节以**斜体**渲染

## CLI 用法

```
read-bin [文件] [选项]
```

| 参数 | 说明 |
|------|------|
| `--dump` | 纯文本 hex dump 输出（不进 TUI） |
| `--copy` | 临时文件快照，外部修改不可见 |
| `--track` | 每 50ms 轮询文件变化 |
| `--inotify` | inotify 事件驱动跟踪（Linux/Android） |
| `--immediate`, `--imm` | 立即模式：每次编辑直写磁盘 |
| `--lock none` | 不锁文件（默认） |
| `--lock 4k` | fcntl range lock 当前 4K 页 |
| `--lock full` | flock(LOCK_SH) 全文锁 |
| `--lock-4k` | 同 `--lock 4k` |
| `--lock-full` | 同 `--lock full` |
| `--unlock` | 同 `--lock none` |
| `-h`, `--help` | 显示帮助 |

### 示例

```bash
read-bin data.bin                   # TUI 模式打开
read-bin data.bin --dump            # 纯文本 hex dump
read-bin data.bin --copy --lock 4k  # 快照 + 4K 锁
read-bin log.bin --inotify          # inotify 跟踪
read-bin data.bin --immediate       # 编辑立即写盘
```

## 顶栏

格式：

```
[1.2MB-icT] *filename.ext
```

- **`[filesize]`** — 当前文件大小（人类可读单位）
- **`-mods`** — 活跃的模式标志（全默认时省略）
- **`*`** — 有未保存修改时显示
- **文件名** — 过长时中间 `...` 省略，保留扩展名

### 模式标志

| 标志 | 含义 |
|------|------|
| `i` | 立即模式（直写磁盘） |
| `f` | 全文锁（`flock`） |
| `4` | 4K 页锁（`fcntl`） |
| `t` | 跟踪模式（50ms 轮询） |
| `T` | inotify 跟踪（事件驱动） |
| `c` | 副本模式（临时文件快照） |

点击顶栏 **`[filesize]`** 或按 **`M`** 打开模式菜单。

## 模式菜单

运行时切换模式，无需重启：

| 项目 | 可切换 | 说明 |
|------|--------|------|
| Track | ✅ | 每 50ms 轮询文件变化 |
| Inotify | ✅ | inotify 事件驱动跟踪（与 Track 互斥） |
| Immediate | ✅ | 直写磁盘 |
| Copy | ❌（仅启动时） | 临时文件快照 |
| ───── | — | 分隔线 |
| Lock: none/4k/full | ✅ | 循环切换锁模式 |

除 **Copy** 外所有模式均可运行时切换。

## 显示模式

点击状态栏 `[ASCII]` 打开下拉菜单，或按 `m` 循环切换。

| 模式 | 说明 |
|------|------|
| ASCII | 可打印字符原样显示，不可打印显示为 hex |
| HEX | 所有字节显示为两位十六进制 |
| UTF-8 | 解码显示 UTF-8 字符 |

### 色彩模式

按 `n` 键循环切换，或在下拉菜单中选择。

| 模式 | 说明 |
|------|------|
| off | 无色彩背景 |
| 256 | 字节背景 = 终端调色板色 |
| RGB | 邻居字节背景：R=前一字节，G=当前，B=后一字节 |
| HSL | 邻居字节背景：H=前，L=当前，S=后 |
| Gray | 灰阶背景：RGB(v,v,v)，v=字节值 |
| Heat | 热力图：黑→蓝→红→黄→白 |
| hsl | 位分解 HSL：高4位=色相，中2位=亮度，低2位=饱和 |
| rgb | 位分解 RGB：RR_GGGG_BB (2:4:2 位) |

所有色彩模式自动适配前景色（根据背景亮度选择黑/白）。

## 搜索

| 按键 | 说明 |
|------|------|
| `f` | hex / nibble 模式搜索 |
| `F` | 纯 UTF-8 字符串搜索 |
| `ESC` | 清除搜索高亮 |

### 搜索输入格式

- **精确 hex**：`48656c6c6f`
- **Nibble 通配**：`4x`（高 nibble=4，低任意）
- **Nibble 范围**：`[0-3]f`（高 nibble 0-3，低=f）
- **双范围**：`[A-F][0-3]`（两个 nibble 都在范围）
- **任意字节**：`z`（= 两个任意 nibble）

### 搜索导航

搜索模式下，所有导航键保持各自步长，跳到目标页第一个匹配。

| 按键 | 步长 | 动作 |
|------|------|------|
| `↑` / `↓` | 逐个匹配 | 上/下一个匹配 |
| `←` / `→` | ±1 页 | 目标页第一个匹配 |
| `J` / `K` | ±1 屏 | 目标页第一个匹配 |
| `PGUP` / `PGDN` | ±半屏 | 目标页第一个匹配 |
| `H` / `L` | ±16 页 | 目标页第一个匹配 |
| `O` / `P` | ±256 页 | 目标页第一个匹配 |
| `HOME` | 到开头 | 文件中第一个匹配 |

目标页无匹配时继续向后扫描。

## 编辑模式

按 `i` 进入编辑模式，按 `ESC` 退出。

| 按键 | 动作 |
|------|------|
| `←` `→` `↑` `↓` | 移动光标（跨页） |
| `0`-`9` `a`-`f` | 编辑 nibble（HEX 模式） |
| 任意字符 | 编辑字节（ASCII/UTF8 模式） |
| `Enter` | 插入换行 `\n`（ASCII 模式） |
| `Tab` | 插入制表 `\t`（ASCII 模式） |
| `Alt`+`↑` | 字节值 +1（0xFF 时不变） |
| `Alt`+`↓` | 字节值 -1（0x00 时不变） |
| `Ctrl+Z` | 撤销 |
| `Ctrl+Y` | 重做 |
| `Ctrl+K, R` | 还原光标字节到原始值 |

## 颜色配置

颜色可通过工作目录下的 `color.yaml` 自定义。支持命名色（`red`、`green`、`blue` 等）和 RGB 格式 `[r, g, b]`。

特性：
- `fg: auto` — 根据背景亮度自动选择黑/白前景
- 终端调色板同步 — 读取 `~/.termux/colors.properties` 保持主题一致

参见 `rust/color.yaml` 示例配置。
